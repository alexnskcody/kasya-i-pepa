# Запуск игры

Сборки нет — игра отдаётся как есть любым статическим сервером.

## Запуск

```bash
cd /Users/Shared/SharedWork/ProjectsClaude/PetsMedia/pets-game
python3 -m http.server 8080
```

Запускать в фоне (run_in_background), затем открыть в браузере:

- Обычный режим: http://localhost:8080/
- С отладочной плашкой: http://localhost:8080/?debug=1
  (показывает версию, время сборки и состояние аудио-подсистемы)

Открыть страницу с машины: `open "http://localhost:8080/?debug=1"`.

## Смоук-проверка без браузера

Автотестов нет. Минимум, который можно проверить из терминала:

```bash
curl -sI http://localhost:8080/ | head -1                 # 200
curl -sI http://localhost:8080/js/main.js | head -1        # 200
curl -sI http://localhost:8080/audio/meow.m4a | head -1    # 200, фолбэк-звук на месте
```

Основная верификация — руками в браузере: потыкать питомцев, погладить,
переключить режимы, убедиться, что консоль без ошибок.

## Проверка звука

- Если менялись рецепты звука — перегенерировать файлы:
  `python3 tools/gen_audio.py` (создаёт и .wav, и .m4a через afconvert).
- Поведение на iOS (Lockdown Mode, разблокировка по жесту) с десктопа не
  воспроизводится — проверять на реальном iPhone по live-адресу после пуша.

## Остановка

Убить фоновый процесс сервера (или `pkill -f "http.server 8080"`).
