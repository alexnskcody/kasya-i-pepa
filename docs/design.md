# «Кася и Пепа» — дизайн-док детской браузерной игры

Дата: 2026-07-05. Статус: утверждён автономно (сессия без интерактива), реализуется сразу.

## Идея

Живая «игрушка-террариум» с двумя питомцами, срисованными с реальных фото из папки:

- **Кася** — кошка, шоколадная бурма: тёмно-шоколадная шерсть, светло-карамельная грудка,
  большие уши с розоватой изнанкой, огромные жёлто-зелёные глаза, тёмная «маска» на морде.
  Характер по фото: валяется пузом кверху, вылизывается (иногда с высунутым языком),
  сидит «булочкой», лезет куда не надо.
- **Пепа** — пёс, чёрный цвергшнауцер: кудрявая чёрная шерсть, борода и брови,
  висячие треугольные уши, зебровый ошейник, хвост-морковка. Характер по фото: сидит и
  смотрит просящим взглядом (умеет служить, трогает лапой), спит плашмя на ковре, всё нюхает.

Целевая аудитория — дети; никаких проигрышей, текста по минимуму, всё через картинки,
звуки и крупные кнопки. Работает в браузере телефона (портрет и ландшафт), офлайн, без зависимостей.

## Ключевые сценарии

1. **Идлы (сами по себе):**
   - Кася: сидит → валяется на спине (машет лапами) → вылизывается → спит калачиком →
     бродит → снова сидит. Уши дёргаются, хвост ходит, глаза моргают, зрачки следят за пальцем.
   - Пепа: сидит и смотрит на игрока просящими глазами (склоняет голову, поскуливает,
     иногда «служит» как на фото) → спит на боку (Zzz) → нюхает пол → бродит.
2. **Реакции на тык:** кот — «мяу!», подскок, при спаме — прижатые уши и убегает;
   пёс — «гав!», бешеное виляние хвостом, кувырок на спину «почеши пузо».
3. **Поглаживание (провести пальцем):** сердечки; кот жмурится и мурчит (тарахтит),
   пёс блаженно вываливает язык и стучит задней лапой.
4. **Погоня (фирменная сценка):** Кася крадётся к дремлющему Пепе → **БАП** лапой
   по носу → «?!» → погоня с пылью и аниме-линиями скорости → Пепа ловит → комическое
   облако потасовки (торчат лапы, звёзды, «мяв!!», «р-р-р!» — пёс треплет кота, кот
   отбивается) → Кася вырывается и взлетает на когтеточку → отряхивается, Пепа гордо
   виляет внизу → пауза → кот слезает и через время снова пристаёт. Всегда безобидно и смешно.
5. **Игра за персонажа:**
   - **За Касю:** тап по полу — идти; рядом с псом кнопка «БАП!» — задирается, начинается
     погоня, убегай тапами, доберись до когтеточки (салют!). Поймали — весёлая потасовка и побег.
   - **За Пепу:** кот сам дразнится; кнопка «ГАВ!» и погоня за котом тапами; догнал —
     тапай быстро в облаке потасовки (звёзды летят), потом кот сбегает. Кнопка «служить» —
     просящий взгляд, за это дают вкусняшку.
   - **«Вместе» (по умолчанию):** оба живут сами, ребёнок тыкает, гладит, кормит, кидает мячик.
6. **Мячик:** толкни — покатится; Пепа приносит игроку (апорт!), Кася подкарауливает и цапает.
7. **Вкусняшка:** кнопка-печенька — падает угощение, ближайший питомец бежит и хрустит.
8. **Дружбометр:** сердце сверху наполняется от любых взаимодействий; при заполнении —
   конфетти и сценка «нос-к-носу» с большим сердцем. Просто радость, без прогрессии.

## Технические решения

- **Стек:** чистые HTML+CSS+JS, без сборки и CDN; классические `<script src>` — работает с `file://`.
- **Сцена:** один SVG `viewBox="0 0 1200 1200"` с `preserveAspectRatio="xMidYMid slice"` —
  заполняет экран в любой ориентации; видимая область считается в JS, ИИ и спавны кламплены к ней.
  Пол — полоса y≈650…1000 с псевдоглубиной (масштаб 0.78…1.12 по y).
- **Персонажи:** рисуются кодом (js/art-cat.js, js/art-dog.js) как наборы SVG-поз
  (sit/walk/lie-back/groom/sleep/sneak/beg/sniff…), внутри каждой позы анимируемые группы:
  глаза (моргание scaleY), зрачки (следят за указателем), уши, хвост, голова, язык.
  Переключение поз — data-атрибут, микродвижения — CSS-keyframes, перемещение — JS-луп.
- **Звук:** Web Audio, всё синтезировано (мяу/мяв, гав, мурчание-LFO, скулёж, боинг, шаги,
  хруст, храп, колокольчики, фанфары). Разблокировка по первому тапу (сплэш «Играть!»),
  кнопка мьюта, состояние в localStorage.
- **Ввод:** Pointer Events (мышь и палец одинаково), `touch-action:none`, без зума/выделения.
- **Производительность:** только transform/opacity-анимации, лимит частиц, один rAF-луп.

## Архитектура

| Файл | Ответственность |
|---|---|
| index.html | сцена-комната (фон, ковёр, окно, диван, когтеточка), сплэш, HTML-UI |
| css/style.css | раскладка, кнопки, keyframes микродвижений и частиц |
| js/audio.js | синтезатор звуков, мьют |
| js/fx.js | частицы и пузыри реплик в SVG-слое |
| js/art-cat.js, js/art-dog.js | SVG-позы персонажей |
| js/pet.js | класс Pet: позы, перемещение, моргание/зрачки, реакции |
| js/director.js | идл-ИИ обоих, сценарий погони, мячик, вкусняшки, дружбометр |
| js/main.js | загрузка, режимы, ввод, кнопки, ресайз |

## Отсечено (YAGNI)

Мета-прогрессия, магазины, счёт, таймеры, день/ночь, кормление мисками, сохранения,
мультиплеер, спрайтовые движки. Имена персонажей — константы, легко поменять.

## Самопроверка спека

Плейсхолдеров нет; сценарии не противоречат друг другу (погоня блокирует идлы через
директора); объём — один связный «той»-луп, декомпозиция не нужна; двусмысленностей
не осталось (поимка = всегда потасовка-облако → побег кота, проигрыша не существует).
