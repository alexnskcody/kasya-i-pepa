# Кася и Пепа — инструкции для сессий Claude

## Команды

```bash
python3 -m http.server 8080        # локальный запуск → http://localhost:8080
python3 tools/gen_audio.py         # регенерация audio/*.wav и *.m4a (afconvert — есть в macOS)
```

Сборки, тестов и package.json нет — это осознанно, не добавлять.

## Архитектура

- Чистые HTML+CSS+JS, ноль зависимостей; хостится статикой на GitHub Pages
  (`.nojekyll` обязателен, live: alexnskcody.github.io/kasya-i-pepa).
- `index.html` — сцена; `js/art-*.js` — SVG-позы персонажей; `js/pet.js` —
  движение/взгляд; `js/director.js` — поведение и сценки; `js/audio.js` —
  звук; `js/fx.js` — эффекты; `js/main.js` — вход. Дизайн-док: `docs/design.md`.
- Звук в норме — синтез через Web Audio; файлы `audio/*` — фолбэк для iOS
  Lockdown Mode, генерируются `tools/gen_audio.py` (руками не редактировать).
- `?debug=1` — отладочная плашка (версия, время сборки, состояние аудио).

## Инварианты (нарушение = баг)

- Коммиты, доки и все артефакты — НА РУССКОМ. Не переводить на английский.
- iOS Lockdown Mode: Web Audio нет, WAV/PCM не декодируется — фолбэк-звук
  только AAC (`.m4a`); выбор расширения через `canPlayType('audio/mp4')`.
- Фолбэк-звуки — настоящие HTTP-файлы из `audio/`, НЕ data:-URI и не
  blob-спрайты (Lockdown их не играет — уже наступали, см. историю коммитов).
- Тихий разблокировочный аудио-луп включается только на iOS; на других
  платформах его быть не должно.
- Игровой цикл не должен умирать от ошибок аудио — звук всегда best-effort.

## Workflow

- Нетривиальная работа идёт через flow-плагин: /flow:spec → одобрение
  владельца → /flow:implement (свежая сессия) → /flow:accept → приёмка.
  Тривиальные правки (диф в одно предложение, без зависимых мест) — мимо
  цикла.
- Если владелец просит нетривиальное, не упомянув flow, — предложить
  /flow:spec одной строкой, без лекций. Отказался — работаем как обычно.
- Worktree-сессии: постоянной памяти нет — всё несут этот файл и openspec/;
  по завершении фазы мержить в main.
- Если flow-плагин недоступен (облачная VM, чужая машина) — повторить форму
  цикла вручную: openspec-изменение → одобрение → реализация на ветке →
  проверка запуском → ревью свежим контекстом → приёмка.
- Проверка перед приёмкой — запуском (скилл run-game): автотестов нет,
  верификация ручная в браузере, обязательно со сценарием `?debug=1`.
- Не коммитить и не пушить без явной просьбы.
